## html 태그 같은 곳에 style 태그나 기타 여러 속성이 있다면 속성를 적는 순서 같은 것이 있는지? 국제 표준 같은거 말이야

검토됨 20개의 소스

HTML 태그에 `style` 을 비롯한 여러 속성을 작성할 때, 기술적으로 브라우저가 인식하는 데 영향을 주는 공식적인 국제 표준 순서는 없습니다. 웹 표준을 정하는 W3C나 WHATWG에서도 태그 내 속성의 순서를 강제하지 않습니다.

하지만 코드의 가독성과 유지보수 향상을 위해 많은 개발자나 기업은 자체적인 코딩 컨벤션(규칙)을 정해 따릅니다. 이는 협업의 일관성을 높이는 데 도움을 주지만, 필수는 아닙니다.

## 일반적인 속성 순서 컨벤션

일반적으로 권장되는 속성 작성 순서는 다음과 같습니다.

- `class`: 가장 자주 사용되므로 보통 맨 앞에 작성합니다.
- `id`, `name`: 페이지 내에서 고유한 식별자 역할을 합니다.
- `data-*`: 사용자 정의 데이터를 저장하는 속성입니다.
- `src`, `href`, `action`: 리소스의 경로를 지정하는 속성입니다.
- `width`, `height`, `alt`: 콘텐츠의 크기나 대체 텍스트를 나타냅니다.
- `role`, `aria-*`: 웹 접근성 관련 속성입니다.
- `style`: 인라인 스타일은 다른 스타일 규칙보다 우선순위가 높고, 구조와 스타일을 분리하는 원칙에 따라 보통 마지막에 작성하는 것을 권장합니다.[velog](https://velog.io/@cindycho0423/JavaScript%EC%8A%A4%ED%83%80%EC%9D%BC-%EB%8B%A4%EB%A3%A8%EA%B8%B0)

## CSS 적용 우선순위

속성의 작성 순서와는 다른 개념으로, 어떤 CSS 스타일이 적용될지를 결정하는 '우선순위(Specificity)' 규칙이 존재합니다. 여러 스타일이 하나의 요소에 중복으로 적용될 때 이 규칙에 따라 최종 스타일이 결정됩니다. 우선순위는 다음과 같습니다.[naver +2](https://blog.naver.com/lux-00/221466770614)

1. `!important` 선언
2. 인라인 스타일 (`style` 속성)
3. ID 선택자 (`#id`)
4. 클래스(`.class`), 속성(`[type="text"]`), 가상 클래스(`:hover`) 선택자
5. 태그(`<h1>`) 및 가상 요소(`::before`) 선택자
