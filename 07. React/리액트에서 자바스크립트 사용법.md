Q. JSX 에서 중괄호를 2번을 써서 자바스크립트를 사용하는 환경이 나오는데 왜 그런걸까?

## 원리 설명

### 1. 첫 번째 중괄호 `{}`

- **JSX 문법**: "여기서부터 JavaScript 표현식이 시작된다"는 의미
- JSX 안에서 JavaScript 값을 사용하려면 반드시 `{}`로 감싸야 함 (쉽게 말해 자바스크립트를 사용하겠다는 이니시에이팅)

### 2. 두 번째 중괄호 `{}`

- **JavaScript 객체 리터럴**: 실제 JavaScript 객체를 만드는 문법
- 객체를 만들 때 사용하는 일반적인 JS 문법

## 구체적인 예시

```jsx
// style 속성에 객체를 전달하는 경우
<div style={{ color: 'red', fontSize: '16px' }}>
  //      ^                                  ^  <- JSX 중괄호
  //       ^                                ^   <- 객체 리터럴 중괄호
```

이를 분해하면:

```jsx
// 1단계: JavaScript 객체 만들기
const styleObject = { color: 'red', fontSize: '16px' };

// 2단계: JSX에서 그 객체를 사용하기
<div style={styleObject}>
```

## 왜 이런 구조인가?

JSX의 속성에는 **문자열**이 아닌 **JavaScript 값**(객체, 배열, 함수 등)을 전달해야 할 때가 많음.

```jsx
// 문자열은 따옴표로
<div className="container">

// 객체는 중괄호 안에 중괄호로
<div style={{ margin: 10 }}>

// 배열도 중괄호로
<div data-items={[1, 2, 3]}>

// 함수도 중괄호로  
<button onClick={() => alert('hi')}>
```

정리하면, **중괄호가 겹치는 건 JSX 문법(외부)과 JavaScript 객체 리터럴(내부)이 만나서 생기는 자연스러운 결과임. 이유 없이 그런 게 아니라, 두 개의 다른 문법 규칙이 조합된 형태라고 보면됨.
